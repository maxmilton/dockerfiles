# SPEC

## ¬ßG GOAL

Monorepo build system for Max's personal Docker images. `make` builds/tests/publishes each `<name>/Dockerfile` to `ghcr.io/maxmilton`, rebuilding only images with stale Alpine packages.

## ¬ßC CONSTRAINTS

- base = `alpine:<ver>@sha256:<digest>`, pinned by tag + digest. all active images same ver (currently 3.24).
- Makefile generic, dockerfile-driven. ‚ä• hardcode image names ‚Äî discover via glob `*/Dockerfile`. dir name = image name.
- `.extra/` images ‚ä• built by default & ‚ä• in CI.
- shell: browser launchers = bash, `docker-entrypoint.sh` = POSIX `sh`. all `.sh` ! pass `shellcheck -ax`.
- registry prefix ! via `REGISTRY` env (default `ghcr.io/maxmilton`).
- adapted from philpep/dockerfiles.

## ¬ßI INTERFACES

- cmd: `REGISTRY=local make all` ‚Üí build every image
- cmd: `make <name>` ‚Üí build one image (chromium|firefox|gcloud|net-block|net-cache|net-tools)
- cmd: `make run <name>` / `make exec <name>` ‚Üí build then run / open shell
- cmd: `make check <name>` ‚Üí run `check_update.sh` in built image, exit‚âÝ0 if apk/apt upgradable
- cmd: `make checkrebuild all` (aka `make ci`) ‚Üí build + rebuild `--no-cache` if stale
- cmd: `make publish` ‚Üí `checkrebuild push all`
- script: `check_update.sh <image>` ‚Üí green "up-to-date" | red "needs update" + exit‚âÝ0
- script: `should_publish.sh <name> [last-published-sha]` ‚Üí exit 0 publish | exit 1 skip (‚ü∫ `.apk-changed-<name>` marker exists | git diff `<name>/` vs last-sha nonempty | last-sha empty)
- script: `<name>/launch.sh` ‚Üí hardened host-side `docker run` wrapper (chromium, firefox); honors `REGISTRY`
- file: `<name>/Dockerfile` ‚Üí one per image; `FROM $REGISTRY/<other>` ! for inter-repo base dep
- env: `REGISTRY` optional; `DOCKER_BUILDKIT=1` exported by Makefile
- env: `LAST_PUBLISHED_SHA` optional, set by publish.yml via `gh run list` (last successful publish run's headSha)
